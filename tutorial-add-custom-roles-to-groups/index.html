    Add custom roles to groups¶

    Lots of games and apps build social communities with groups. These groups are made up of admins and members. Sometimes it's useful to have more than two roles of authority within a group.

    In this tutorial we'll cover how to set max group size limits, add new custom roles for members, disable some client messages, and add validation rules which fit your gameplay or app experience.

    Builtin roles¶

    Both "admin" and "member" are builtin roles.

    An admin can accept new members, promote a member as an admin, demote a user, or kick a member from the group. A member can join a group and with other members participate in group chat. You can read more about the capabilities in groups.

    Add a new role¶

    We'll add a new role with permissions in between an admin and a member. The new "manager" role will be capable of:

    • Add or reject new members.
    • Kick a member from the group.
    • Promote a member as a manager.

    An admin will be able to create a manager. These rules can be built with server-side code.
